# Professor Rulebook

This folder is the English operational version of the lecturer PDFs in:

- `Important Rolse/CodingStandards_NoSC.pdf`
- `Important Rolse/Ex_Submit_Instructions_DN_MTA_26B.pdf`
- `Important Rolse/VisualStudio_Instructions.pdf`

The goal is simple: make the rules easy to follow without changing their meaning.

## Non-Negotiable Rules

1. Follow the lecturer PDFs as the source of truth.
2. Use `Visual Studio for Windows` for course work.
3. Create a proper `solution`, then add the exercise projects into it.
4. Follow the coding standards exactly:
   meaningful names, correct prefixes, braces, tabs for indentation, spacing rules, and method structure.
5. Keep the solution clean:
   no unused code, no unnecessary generated content, no forbidden folders in the final submission.
6. Use the exact naming format required for the solution, project, and zip.
7. Before submission, delete generated folders such as `bin`, `obj`, `.git`, and `packages`.
8. Submit in the exact way required by the lecturer:
   Google Form first, email fallback if needed, and exact email subject format when emailing.
9. After review, inspect the checked code comments and follow the recheck/question email format exactly.
